# Cloud deployment (roadmap step 10 — Render free web service + Groq API).
#
# Deliberately excludes what the cloud API never needs: docling/reportlab
# (ingestion is a local-only step — you ingest on your laptop against the
# cloud Qdrant/Neo4j, the API only serves QUERIES), celery (no worker
# process on a single free web service), streamlit (the UI deploys
# separately to Streamlit Community Cloud, see requirements-ui.txt), and —
# after the deployment pivot below — torch/transformers/Ollama entirely.
#
# Pivot history: originally planned to self-host Ollama + the fine-tuned
# GGUF in the cloud. Measured this project's own retrieval+guardrail stack
# (NLI grounding + dense/sparse embedding + reranker) at 719MB RSS with
# torch loaded — before adding a multi-GB Ollama process on top. No no-card
# host offers that much RAM. Fix: generation now calls Groq's free hosted
# API (LLM_PROVIDER=groq) instead of a local model, and NLI grounding is
# disabled for this deployment (GROUNDING_ENABLED=0) — dropping torch
# brings the measured footprint to 198MB, comfortably inside Render's free
# 512MB tier. Full detail + the tradeoff being made explicit: RUNME §15.
